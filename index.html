양방향 로그인 싱크 (Vault-Sync)

라이브리에 로그인하면 언론사에도 로그인됩니다.
단절되었던 사용자 경험을 연결하고, 데이터 주권을 확보하세요.

주요 기능 보기 기술 가이드

사용자 경험(UX) 혁신

댓글을 쓰려다 로그인이 풀려 이탈하는 독자는 이제 없습니다. 한 번의 로그인으로 댓글과 기사 읽기 경험이 끊김 없이 이어집니다.

데이터 주권 확보

소셜 로그인 사용자도 언론사의 자체 회원 데이터로 안전하게 매핑됩니다. 미래 데이터 전략을 위한 핵심 자산을 확보하세요.

신규 회원 유입 가속

댓글 참여를 위해 유입된 소셜 사용자를 자연스럽게 언론사 자체 회원가입으로 유도하여 충성 독자층을 넓힙니다.

유연한 기술 구현 방식

고객사의 개발 환경과 니즈에 맞춰 최적의 연동 방식을 지원합니다.

RECOMMENDED (서울신문 적용)

A. 자체 회원 연동 수정

Fast Track

기존 회원 시스템의 로그인 세션(Cookie/LocalStorage) 정보를 라이브리 Vault와 안전하게 공유하는 방식입니다.

  • 빠른 도입: 개발 공수를 획기적으로 단축
  • 레거시 호환: 기존 회원 시스템 구조 변경 불필요
  • 검증된 보안: 서울신문 도입을 통해 안정성 입증
localStorage.setItem('livere_auth', token);
// 프론트엔드 작업만으로 연동 가능
STANDARD

B. OIDC 표준 프로토콜

Standard

업계 표준인 OpenID Connect 인증 프로토콜을 통해 인증 정보를 서버 간 교환하는 방식입니다.

  • 표준 준수: OAuth 2.0 / OIDC 기반 인증
  • 높은 확장성: 타 시스템과의 연동성 확보
  • 자체 인증 서버(OP) 구축 필요 (개발 공수 높음)
GET /authorize?response_type=code
// 표준 인증 서버 구축 필요
